## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly and privately. Do not open a public GitHub issue for security-related concerns.

Please report vulnerabilities by contacting the project maintainers through the appropriate private channel, such as a dedicated security email address or GitHub private vulnerability reporting, if enabled.

When submitting a report, please include as much detail as possible, including:

- A clear description of the vulnerability
- Steps to reproduce the issue
- The affected version(s)
- Any proof-of-concept code, logs, or screenshots that may help us validate the issue
- The potential impact of the vulnerability

After a report is received, we will acknowledge receipt within 3 business days. We will then investigate the issue and provide status updates as appropriate throughout the review process.

If the vulnerability is confirmed, we will work on a fix and coordinate responsible disclosure. You may be asked to keep the issue confidential until a patch or mitigation is available.

If the report is determined not to be a security vulnerability, or if it cannot be reproduced, we will let you know with an explanation where possible.

We appreciate your efforts to help keep this project and its users secure.
